When the user clicks "Apply global filter":
  1. Build a list of FilterSpec from the rows where both

       st.session_state['global_filter_stats'] = stats

       st.session_state['cross_cut_skips'] = []
       st.session_state['filtered_results'] = {}

TASK 4 — Per-question multi-filter panel

Inside each single-cut card (section 3), the filter panel 
becomes dynamic. Replace the fixed 5-column layout from 
Day 12 with a flexible system:

UI:
  
  st.markdown("**Filters for this question**")
  
  # Pull the current filter list for this canonical_id
  filter_key = f"filters_{spec.canonical_id}"
  if filter_key not in st.session_state:
      st.session_state[filter_key] = []  
        # list of (q_id, value) tuples
  
  filter_rows = st.session_state[filter_key]
  
  # Display existing filter rows
  for i, (q_id, val) in enumerate(filter_rows):
      cols = st.columns([4, 4, 1])
      with cols[0]:
          new_q = st.selectbox(
              "Filter question",
              options=[("None", None)] + [
                  (f"{q.canonical_id}: {q.question_text[:50]}", 
                   q.canonical_id)
                  for q in eligible_filter_questions
              ],
              format_func=lambda x: x[0],
              index=_find_index(q_id, options),
              key=f"{filter_key}_q_{i}",
              label_visibility="collapsed" if i > 0 else "visible",
          )
      with cols[1]:
          if new_q[1]:
              q = schema.get_question(new_q[1])
              new_val = st.selectbox(
                  "Value",
                  options=[("All values (breakdown)", None)] + [
                      (f"{v}: {label}", v) 
                      for v, label in q.option_map.items()
                  ],
                  format_func=lambda x: x[0],
                  index=_find_value_index(val, options),
                  key=f"{filter_key}_v_{i}",
                  label_visibility="collapsed" if i > 0 else "visible",
              )
          else:
              new_val = (None, None)
      with cols[2]:
          if st.button("✕", key=f"{filter_key}_del_{i}", 
                        help="Remove this filter"):
              filter_rows.pop(i)
              st.rerun()
      
      filter_rows[i] = (new_q[1], new_val[1])
  
  cols_btn = st.columns([2, 2, 4])
  with cols_btn[0]:
      if st.button("+ Add filter", key=f"{filter_key}_add"):
          filter_rows.append((None, None))
          st.rerun()
  with cols_btn[1]:
      if st.button("Apply filters", key=f"{filter_key}_apply",
                    disabled=not any(q for q, v in filter_rows)):
          # Build FilterSpec list, dispatch
          filter_specs = [
              FilterSpec(filter_question_id=q, filter_value=v)
              for q, v in filter_rows if q is not None
          ]
          # Validate: max one breakdown filter
          breakdowns = [
              f for f in filter_specs if f.filter_value is None
          ]
          if len(breakdowns) > 1:
              st.error(
                  "Only one breakdown filter at a time. "
                  "Pick a value for at least all but one of "
                  "the breakdowns.")
          else:
              try:
                  filtered_result = compute_filtered_single_cut(
                      spec.canonical_id,
                      filter_specs,
                      schema,
                      st.session_state['active_df'],  
                        # this is the global-filtered df!
                      st.session_state['log'],
                  )
                  st.session_state['filtered_results'][
                      spec.canonical_id] = filtered_result
                  st.rerun()
              except Exception as e:
                  st.error(f"Filter failed: {e}")

The eligible_filter_questions list = all schema questions 
where option_map is non-empty (categorical with values to 
match against).

CRITICAL: The DataFrame passed to compute_filtered_single_cut 
is st.session_state['active_df'] — which is the GLOBAL-
FILTERED DataFrame, not the original. This is what makes 
the global filter actually applied to per-question filters: 
they layer on top.

Display logic stays the same:
  - If filtered_results[canonical_id] exists → show 
    filtered display (banner + result + checkbox + clear 
    button)
  - Else → show unfiltered single cut
