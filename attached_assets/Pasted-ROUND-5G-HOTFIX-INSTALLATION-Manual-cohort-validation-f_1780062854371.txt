ROUND 5G HOTFIX INSTALLATION — Manual cohort validation falls back to uuid/respondent_id/id columns when primary respondent_id_column produces zero matches.

CONTEXT

The previous Round 5G shipped with a validation bug: when the survey's primary respondent_id_column was "record" (sequential integers) but the uploaded manual cohort contained uuid strings, validation produced "0 valid, 472 not found" for every uuid even though all 472 existed in the raw data's uuid column.

This hotfix adds a TIER 2 fallback. Validation tries the primary respondent_id_column first; if zero matches AND there are invalid uuids, retries against uuid / respondent_id / id columns (case-insensitive) and uses the first one that produces matches.

The resolved column is stored in SegmentDefinition.manual_cohort_id_column (new field, defaults to None) so mask construction uses the same column.

FILES BEING UPLOADED

1. src/models.py — adds manual_cohort_id_column field to SegmentDefinition with default None. All other fields preserved.
2. src/io.py — adds id_column parameter to parse_manual_cohort_workbook; embedded W&L parser also implements TIER 2 fallback.
3. src/outcome_segmentation.py — _build_segment_masks uses manual_cohort_id_column when set.
4. app.py — two-tier validation in manual cohort upload section with app.info() message on fallback.
5. tests/test_manual_cohort_upload.py — replaces existing. Includes new fallback tests.
6. tests/test_app_manual_cohort_ui.py — replaces existing.
