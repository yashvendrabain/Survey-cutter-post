ROUND 5L HOTFIX 3.1 — Narrow DATAMAP_KEYWORDS to remove false-positive matches.

PROBLEM

Hotfix 3 added trial-parse verification that correctly catches false-positive adapter matches. But the underlying root cause — DATAMAP_KEYWORDS being too broad — still wastes cycles. Replit's diagnostic confirmed: "Question_Metadata" matches "metadata", causing the adapter to attempt parsing the wrong sheet, fail, then route to wizard.

The trial-parse catches this in production, but it's defense-in-depth, not root-cause fix.

Also: if a workbook had a sheet legitimately named "Variables" or "Dictionary" with structured content that parsed to 2+ questions, the trial parse would SUCCEED on the wrong sheet, producing garbage output.

FIX

In src/adapters/compact_two_column.py and src/adapters/bcn_multicolumn.py, narrow DATAMAP_KEYWORDS to ONLY strong unambiguous tokens:

    DATAMAP_KEYWORDS = (
        "datamap",       # matches: "Data map", "DataMap", "data_map", "Datamap"
        "codebook",
        "schema",
    )

REMOVE:
- "dictionary"
- "questions" 
- "questionnaire"
- "variables"
- "metadata"
