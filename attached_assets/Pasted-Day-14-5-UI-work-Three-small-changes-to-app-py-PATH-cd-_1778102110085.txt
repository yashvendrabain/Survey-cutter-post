CHANGE 2 — Add a copy button to every inline cross-cut 
preview table

Streamlit's st.dataframe has a built-in copy/download UI on 
hover (the small toolbar in the upper right of the rendered 
table — it appears automatically). Confirm this is enabled 
by NOT setting hide_dataframe_toolbar or any toolbar=False 
prop.

If the toolbar isn't appearing, the issue is likely an old 
Streamlit version. Run:
  streamlit --version

If less than 1.30, the toolbar may need explicit enablement 
or a Streamlit upgrade. If you can't upgrade, add a manual 
"Copy as TSV" button below each table:

  if st.button(
      "Copy as TSV", 
      key=f"copy_{result.cross_cut_id}",
  ):
      tsv = df.to_csv(sep="\\t", index=True)
      st.code(tsv, language="text")
      st.caption(
          "Select all and copy. Paste into Excel or "
          "Google Sheets.")

The st.code block has a built-in copy button on its 
upper-right hover that the user can click. Document this 
in the caption so the user knows.


CHANGE 3 — Add a percentage toggle to each inline cross-cut 
preview

Below the existing cross-cut preview table, add a small 
radio or selectbox that lets the user re-render the table 
in percentages without re-running the cross cut. The 
underlying CrossCutResult already has counts, row_pct, and 
col_pct in result_table from the engine — this is purely a 
display swap.

  display_mode = st.radio(
      "Display",
      options=["Counts", "Row %", "Column %"],
      horizontal=True,
      key=f"preview_mode_{result.cross_cut_id}",
  )

When the radio is on "Row %" or "Column %":
  - Build the DataFrame from result.result_table["row_pct"] 
    or result.result_table["column_pct"] respectively (these 
    are nested dicts of floats 0.0-1.0)
  - Format cells as percentages: use 
    df.style.format("{:.1%}") and pass to st.dataframe via 
    its data parameter

For Counts mode, render as before with integer formatting.

This applies only to CROSS_TAB analysis type. For 
SEGMENT_PROFILE, GROUP_COMPARISON, and EXPECTED_VS_REALIZED, 
the radio doesn't appear (counts is the only meaningful 
display).


CHANGE 4 — Multi-select inline preview cleanup

In _preview_segment_profile (when target type is multi-
select) and anywhere else multi-select results are rendered 
inline, drop any rows that show "unchecked" or "not 
selected" counts. Only show the "selected" rows.

Each multi-select row in the inline preview should be:

  Sub-column | Label | Selected count

Sorted by Selected count descending. The "unchecked" 
information stays in the underlying data (audit trail 
preserves it) but is never shown to the user.
