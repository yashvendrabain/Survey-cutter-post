TASK 1 — Extend app.py with cross cut UI

After the existing single-cut results section in app.py, 
add a new "Cross Cuts" section that appears only after 
single cuts have completed (st.session_state.run_complete 
is True).

The section has THREE parts:

PART A — Manual cross cut builder

A 3-column layout:
  Col 1: st.selectbox "Question 1" — populated from 
         schema.analysis_eligible_questions(), label format 
         "{canonical_id} — {truncated text}"
  Col 2: st.selectbox "Question 2" — same population
  Col 3: st.selectbox "Analysis type" — populated based on 
         the types of Q1 and Q2 selected:
         - Both SINGLE_SELECT or DEMOGRAPHIC_OR_SEGMENT 
           → CROSS_TAB available
         - Either is SINGLE_SELECT/DEMOGRAPHIC_OR_SEGMENT, 
           the other anything 
           → SEGMENT_PROFILE available
         - One is DIRECT_NUMERIC, other is 
           SINGLE_SELECT/DEMOGRAPHIC_OR_SEGMENT 
           → GROUP_COMPARISON available
         - Both are DIRECT_NUMERIC 
           → EXPECTED_VS_REALIZED available
         If no compatible types, dropdown is disabled 
         with a help message.
         
         GUARDRAILS for Day 9:
         - Multi-select questions are NOT offered for 
           CROSS_TAB (multi-select cross-tab not yet 
           implemented)
         - Allocation questions are NOT offered for 
           GROUP_COMPARISON (semantics need refinement)
         These guardrails prevent the user from triggering 
         the issues Replit flagged in Day 8 review.

  Below the dropdowns:
  - For SEGMENT_PROFILE: a 4th selectbox appears asking 
    which value of Q1 to filter on, populated from Q1's 
    option_map.
  - "Run cross cut" button (st.button), enabled only when 
    a valid combination is selected.

  On click:
  1. Build a CrossCutSpec from the selections
  2. Call compute_cross_cuts([spec], schema, df, log)
  3. Append result to 
     st.session_state['cross_cut_results']
  4. st.rerun() to refresh the display

PART B — Suggested cross cuts (opt-in)

Above Part A, an opt-in checkbox:
  st.checkbox("Show suggested cross cuts", value=False, 
              key="show_suggestions")

When checked:
  - Call src.cross_cut_suggestions.suggest_cross_cuts(
      st.session_state.schema)
  - Display each suggestion as a row with: title, reason, 
    "Run this" button
  - On click: same as Part A run flow

PART C — Cross cut results display

Below Part A, an expander or section showing every cross 
cut in 
st.session_state['cross_cut_results']:
  - Title and analysis type
  - st.dataframe rendering the result_table (you'll need 
    a small helper to convert each result_table dict into 
    a flat DataFrame for display — different for each 
    analysis_type)
  - Remove button per result (removes from session_state)

For now, the cross cut results don't need to be in the 
downloaded workbook — that's Day 10. Just show them in 
the UI.

INITIALISATION
Add to the session_state init at the top of app.py:
  if 'cross_cut_results' not in st.session_state:
      st.session_state['cross_cut_results'] = []

ERROR HANDLING
Wrap each compute_cross_cuts call in try/except, show 
st.error on failure, do not crash the app.
