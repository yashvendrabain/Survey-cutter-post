BUG 1 FIX — Filter value mismatch (label vs code in raw data)

The issue: Q1 stores "India" as a string in the raw data, 
but the option_map says code=7 → label="India". When the 
user picks "India" from the dropdown, the filter stores 
filter_value=7 (the code) but the raw data has "India" 
(the label). df["Q1"] == 7 returns 0 rows.

Fix strategy: When applying a filter, try matching the raw 
data column by code first. If that produces 0 rows AND the 
column contains string values AND the option_map label 
matches values actually in the column, fall back to 
matching by label instead.

In src/global_filter.py, in the apply_global_filter 
function, replace:
    mask = mask & (df[f.filter_question_id] == f.filter_value)

With:
    col = df[f.filter_question_id]
    val = f.filter_value
    # Try code match first
    code_mask = col == val
    if code_mask.sum() == 0:
        # Try string match against the value directly
        str_mask = col == str(val)
        if str_mask.sum() == 0:
            # Try matching the option label from schema
            # The filter_value_label is passed via the 
            # FilterSpec — we don't have schema here.
            # Use str(val) as the label fallback.
            pass
        else:
            code_mask = str_mask
    mask = mask & code_mask

BUT this is incomplete because we don't have the label 
in the filter when only the code is stored.

BETTER FIX — store the label alongside the code in 
FilterSpec. Update the FilterSpec usage in app.py:

When building FilterSpec from UI selections, store 
filter_value as whatever is ACTUALLY in the raw data 
column for that option. Detect this at filter construction 
time:

  def _resolve_filter_value(schema, df, q_id, raw_value):
      """Return the value to use for filtering.
      
      Checks whether the raw_value (the option code from 
      option_map) actually appears in the raw data column. 
      If not, checks whether the option LABEL appears instead.
      If neither, returns the raw_value unchanged.
      """
      if q_id not in df.columns:
          return raw_value
      col_values = set(df[q_id].dropna().unique())
      
      # If the code exists in the column, use it directly
      if raw_value in col_values:
          return raw_value
      if str(raw_value) in col_values:
          return str(raw_value)
      
      # If the code does not exist, look up the label 
      # from the question's option_map and check if 
      # the label exists in the column
      q_spec = schema.get_question(q_id)
      if q_spec and q_spec.option_map:
          label = q_spec.option_map.get(raw_value)
          if label and label in col_values:
              return label
          # Also try str(raw_value) as key
          label2 = q_spec.option_map.get(str(raw_value))
          if label2 and label2 in col_values:
              return label2
      
      return raw_value  # fallback: return as-is

Place this helper in app.py. Call it when building 
FilterSpec for:
  1. Global filter Apply button
  2. Per-question filter Apply button
  3. Suggestion "Run this" button

In each of these places, wrap the filter_value:
  resolved_value = _resolve_filter_value(
      schema, st.session_state['active_df'], q_id, raw_value)
  FilterSpec(filter_question_id=q_id, 
             filter_value=resolved_value)
