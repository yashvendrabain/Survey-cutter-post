CHANGE 1 — Persist outlier highlighting across display 
mode toggles

Currently when the user switches between Counts / 
Counts+% / % only, the flag is recomputed each time 
which is fine, but the issue is the flags dict is 
computed from the count values and those don't change 
with display mode. Verify _compute_outlier_flags is 
called BEFORE the display_mode radio widget renders, 
and the flags list is passed into _render_sc_table_html 
regardless of which mode is selected.

Specifically: the flags computation must NOT be inside 
any conditional block that depends on display_mode. 
Move it above the radio widget:

  # Compute flags ONCE before display mode selection
  counts = [p.get('count', 0) 
            for _, p in result.distribution.items()]
  flags = _compute_outlier_flags(counts)
  
  # Then the toggle
  display_mode = st.radio(
      "Display",
      options=["Counts", "Counts + %", "% only"],
      index=1,
      horizontal=True,
      key=f"sc_display_{result.question_id}",
      label_visibility="collapsed",
  )
  
  # Pass pre-computed flags into the renderer
  _render_sc_table_html(
      result.distribution, 
      display_mode, 
      result.valid_n,
      flags=flags,  # add flags as a parameter
  )

Update _render_sc_table_html signature to accept 
flags as an optional parameter:

  def _render_sc_table_html(
      distribution: dict,
      display_mode: str,
      valid_n: int,
      flags: list | None = None,
  ) -> None:
      if flags is None:

This ensures the red/amber highlighting is identical 
across all three display modes — the same rows stay 
flagged regardless of whether the user views counts, 
percentages, or both.

Apply the same pattern to MultiSelectResult tables —

toggle, pass them into the renderer.
